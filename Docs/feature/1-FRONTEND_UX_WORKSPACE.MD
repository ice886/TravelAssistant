# FRONTEND UX WORKSPACE

## 1. 功能定位

前端工作台将旅行规划组织为三个连续阶段：描述旅程、收集证据、编辑行程。页面不改变既有 API 和数据模型，而是通过统一视觉语言和状态反馈帮助用户理解当前进度与下一步行动。

## 2. 页面结构

- 顶部服务状态：展示 API、数据库和小红书数据源状态。
- 旅程阶段导航：根据当前 trip 和 research run 自动标记当前阶段；导航可切换已解锁阶段，未满足前置条件的阶段保持禁用。
- 旅行画像：收集目的地、天数、兴趣、预算与同行人信息。
- 多来源研究：启动 Research Agent，展示配置检查、降级、轮次和工具记录。
- 研究资料库：Tavily 网页来源仅展示简体标题和原始来源链接；小红书与高德来源展示简体标题、简体摘要和原始来源链接。provider 类型保留在数据层，不在列表项重复展示。
- 旅行计划：生成首版行程并以紧凑只读活动卡片展示每日安排、时间、地点、交通和费用。
- 旅行计划活动卡片：优先识别“自驾 + 出发/前往/返程/还车/公路”等路线意图，再匹配餐饮、艺术、摄影、自然、地标等活动内容，最后识别火车和步行交通，避免描述中的“拍摄”覆盖驾车路线。

## 3. 设计与交互约束

- 使用 `app/styles.css` 中的语义色彩和布局 Token，不在组件内增加散落的样式值。
- 所有主要异步操作提供等待、成功、错误或禁用反馈。
- 系统错误文案承担系统责任，并提供清晰的恢复方向。
- 单页采用阶段工作台：同一时间只展示“描述旅程”“收集证据”或“编辑行程”中的一个阶段，避免四个面板纵向堆叠造成过度滚动。
- 研究阶段内部使用双栏展示 Research 与来源；桌面端保持双栏，平板和移动端自然重排为单列。
- 每个阶段提供上一步/下一步操作；创建旅行后进入研究阶段，研究完成后进入行程阶段。
- 交互目标最小高度 44px，保持明显键盘焦点，并支持 `prefers-reduced-motion`。

## 4. 当前边界

- 当前状态继续由 `App` React state 和各面板内部 state 管理；第三阶段仅展示旅行计划，不提供前端编辑控件。
- Research 继续采用 3 秒轮询；未引入 React Query 或 Zustand。
- 地图联动、PDF 导出和复杂跨页面导航属于后续功能。
- 来源标题与摘要通过前端 `chinese-conv` 在展示时繁转简，后端保存的原始研究内容不变。
